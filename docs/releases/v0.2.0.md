# Continuation Layer v0.2.0 Preview

Continuation Layer v0.2.0 adds Linux-first experimental interactive Codex terminal wrapping on top
of the v0.1 non-interactive watchdog foundation.

This is a preview release note for final review. Tagging and publishing should happen only after CI
is green on the release commit.

## Completed

- `continuity shell --dry-run` command planning.
- `continuity shell` PTY runner foundation through `node-pty`.
- Interactive Codex command construction through the existing Codex provider adapter.
- Non-TTY guard with clear guidance to use `continuity watch`.
- PTY stdin/stdout pass-through, raw-mode restore, resize forwarding, and cleanup coverage.
- ANSI-stripped rolling-buffer cooldown detection for PTY output.
- Interactive cooldown state recording:
  - `status = cooling_down`
  - `mode = cooldown_resume`
  - `cooldown_detected_at`
  - `next_resume_at`
  - `reset_time_provenance`
  - `.agent/AUTO_SNAPSHOT.md`
  - `interactive_cooldown_detected` sessions event
- User-confirmed graceful pause after interactive cooldown detection.
- Wait until `next_resume_at`, then interactive resume.
- Resume target selection:
  - `codex resume <session_id>` when an explicit session id exists
  - `codex resume --last` as best-effort fallback
- Adoption of existing interactive `cooling_down` state when `continuity shell` is restarted.
- Manual interactive smoke checklist in `docs/SMOKE_INTERACTIVE.md`.

## Safety Boundaries

- No provider-limit bypass.
- No account rotation.
- No fake reset windows.
- No hard kill of Codex by default on interactive cooldown.
- No auto commit.
- No automatic PRs.
- No GUI or dashboard.
- No Claude Code interactive wrapper in v0.2.
- Direct already-running `codex` processes cannot be adopted.

## Known Limitations

- Linux-first only.
- macOS may work but is not guaranteed yet.
- Windows native interactive shell support is not included; WSL is the likely path for Windows users.
- Real interactive smoke testing is manual and not part of CI.
- `codex resume --last` is best-effort and depends on Codex session selection behavior.
- Cooldown visibility depends on Codex TUI output exposing recognizable cooldown text.
- Provider CLI behavior and private session storage may change.
- Package version bump, tagging, and publishing are separate release operations.

## Verification

```sh
git diff --check
```

Manual smoke:

```sh
docs/SMOKE_INTERACTIVE.md
```

## Upgrade Notes

- Existing v0.1 `continuity watch` behavior remains the recommended non-interactive long-task path.
- `continuity shell` is for interactive terminal work and must be used instead of starting `codex`
  directly if cooldown monitoring is required.
- Direct `codex` sessions remain outside Continuation Layer supervision.
